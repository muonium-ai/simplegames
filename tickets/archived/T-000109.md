## Progress Log
- 2026-05-22: Added --autoplay argparse flag to both rubiks_cube.py and rubiks_cube_3d.py. When set, the game scrambles 20 moves then solves automatically on launch, looping forever (scramble -> solve -> scramble -> ...). ESC still quits between iterations because the event loop runs every frame. In rubiks_cube.py the loop is triggered from the post-solve branch after verify_solved_state. In rubiks_cube_3d.py a small state machine in the main loop watches (not animating AND no pending moves) to transition scrambling<->solving. Preserved T-000050 Kociemba center mapping, T-000051 R2 3D fix, T-000056 non-blocking pacing, T-000070 ESC quit, and KMOD_SHIFT/MOUSEBUTTONUP imports. py_compile clean.
