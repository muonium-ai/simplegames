## Goal
When the player clicks a mine in manual mode, show a centered Game-Lost
modal with two buttons — Play New Game and Exit Game — instead of the
text-only "Press R to restart" overlay.

## Acceptance Criteria
- [x] Modal appears in both minesweeper scripts on manual-mode loss.
- [x] "Play New Game" calls reset_game(); "Exit Game" quits cleanly.
- [x] Grid + header clicks are ignored while the modal is up (event loop
      short-circuits with `continue`).
- [x] Autoplay / solver paths skip the modal so the T-000117 auto-restart
      still fires.
- [x] Victory branch unchanged (still uses the existing overlay).

## Notes
- Probability script reuses the existing Button class; solver script uses
  inline pygame.Rect + collidepoint since it has no Button helper.
- Buttons are constructed at fixed window-center coords in __init__ so
  click hit-testing works on the same frame the modal first draws.
- Verified headlessly via /tmp/verify_modals.py: forced game_over=True,
  called draw(), pygame.image.save'd the screen; both screenshots show
  the modal correctly layered over the revealed-mine board.
