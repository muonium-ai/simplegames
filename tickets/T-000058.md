## Goal
Define a uniform in-game quit convention so every game can be exited cleanly from the keyboard without closing the window.

## Acceptance Criteria
- [ ] **ESC** quits the game cleanly from anywhere — title screen, gameplay, pause menu, game-over screen.
- [ ] Clean exit means `pygame.quit()` followed by `sys.exit(0)` (or platform-equivalent for non-pygame games). No half-torn-down state, no lingering audio/timer threads.
- [ ] The existing window-close (`pygame.QUIT` event) path also performs the same clean exit. ESC and clicking the X button should be indistinguishable.
- [ ] A visible "ESC to quit" hint appears on the start screen and on the game-over screen (small text, bottom-right or bottom-center). HUD hint is optional.
- [ ] Where ESC is already bound to something else (e.g., pause / back to menu), prefer **Q** for direct quit and document the chosen key in that game's ticket.

## Notes
- All pygame-based games should handle the keypress inside the same `for event in pygame.event.get()` loop they already use for QUIT:
  ```python
  if event.type == pygame.QUIT or (event.type == pygame.KEYDOWN and event.key == pygame.K_ESCAPE):
      pygame.quit()
      sys.exit(0)
  ```
- Helper pattern: extract a `def quit_game(): pygame.quit(); sys.exit(0)` at module scope if the same logic is needed in multiple loops (start screen + main loop + game-over).
- Non-pygame games (the Swift slidergame, chess-game pure logic) are out of scope.
- This ticket is the **spec** only. Per-game implementation tickets reference this one via `depends_on`.
